# 系统文档

## 1. 系统概述

Hermes Platform 是一个测试管理平台，旨在帮助团队管理测试任务、测试详情和测试记录，提供完整的测试生命周期管理功能。系统包含用户认证、权限管理、测试任务管理、测试详情管理和测试记录管理等核心功能。

## 2. 系统架构

### 2.1 架构设计

系统采用分层架构设计，主要包含以下层次：

1. **API 层**：处理 HTTP 请求，路由管理，请求验证
2. **服务层**：业务逻辑处理
3. **存储层**：数据访问和持久化
4. **模型层**：数据模型定义
5. **中间件**：请求处理中间件
6. **工具层**：通用工具函数

### 2.2 核心组件

| 组件 | 职责 | 位置 |
|------|------|------|
| API 路由器 | 处理 HTTP 请求和路由 | `internal/api/routes.go` |
| 认证服务 | 处理用户认证和权限 | `internal/services/auth_service.go` |
| 测试服务 | 处理测试任务、详情和记录 | `internal/services/test_service.go` |
| 权限服务 | 处理权限管理 | `internal/services/permission_service.go` |
| 数据存储 | 数据库访问和操作 | `internal/repository/` |
| 中间件 | 请求处理和安全 | `internal/middleware/` |
| 配置管理 | 系统配置 | `internal/config/` |

## 4. 核心功能模块

### 4.1 认证模块

**功能**：用户注册、登录、密码修改、个人信息管理

**实现**：
- 使用 JWT 进行身份验证
- 密码加密存储
- 基于角色的权限控制

**关键流程**：
1. 用户注册：验证参数 → 加密密码 → 创建用户 → 返回用户信息
2. 用户登录：验证凭证 → 生成 JWT 令牌 → 返回令牌和用户信息
3. 身份验证：解析 JWT 令牌 → 验证令牌有效性 → 提取用户信息

### 4.2 测试任务管理

**功能**：创建、查询、更新、删除测试任务，获取测试任务进度

**实现**：
- 测试任务 CRUD 操作
- 任务状态和优先级管理
- 任务关联的测试详情和记录查询
- 基于 UUID 的测试任务查询
- 测试任务进度和通过率计算

**关键流程**：
1. 创建测试任务：验证参数 → 生成 UUID → 保存任务 → 返回任务信息
2. 查询测试任务：应用过滤条件 → 分页查询 → 返回任务列表
3. 通过 UUID 查询测试任务：验证 UUID → 查询任务 → 返回任务信息
4. 获取测试任务进度：查询任务 → 计算进度和通过率 → 返回进度信息
5. 更新测试任务：验证权限 → 更新任务信息 → 清除缓存 → 返回更新结果
6. 删除测试任务：验证权限 → 查询任务 → 清除缓存 → 删除任务 → 返回删除结果

### 4.3 测试详情管理

**功能**：创建、查询、更新、删除测试详情

**实现**：
- 测试详情 CRUD 操作
- 与测试任务关联
- 预期结果管理

**关键流程**：
1. 创建测试详情：验证参数 → 关联测试任务 → 保存详情 → 返回详情信息
2. 查询测试详情：根据 ID 查询 → 返回详情信息
3. 更新测试详情：验证权限 → 更新详情信息 → 返回更新结果
4. 删除测试详情：验证权限 → 删除详情 → 返回删除结果

### 4.4 测试记录管理

**功能**：创建、查询、更新、删除测试记录

**实现**：
- 测试记录 CRUD 操作
- 与测试任务和测试详情关联
- 测试状态和结果管理

**关键流程**：
1. 创建测试记录：验证参数 → 关联任务和详情 → 保存记录 → 返回记录信息
2. 查询测试记录：根据 ID 查询 → 返回记录信息
3. 更新测试记录：验证权限 → 更新记录信息 → 返回更新结果
4. 删除测试记录：验证权限 → 删除记录 → 返回删除结果

### 4.5 权限管理

**功能**：基于角色的权限控制

**实现**：
- 权限中间件
- 基于权限的访问控制
- 权限验证

**关键流程**：
1. 权限验证：提取用户角色 → 检查权限 → 允许或拒绝访问

## 6. 系统流程

### 6.1 服务启动流程

1. 加载配置文件
2. 初始化 JWT 配置
3. 连接数据库并自动迁移模型
4. 初始化默认数据（包括管理员账户）
5. 尝试连接 Redis（失败不影响服务启动）
6. 初始化 Gin 路由
7. 注册中间件（日志、错误处理、速率限制）
8. 注册 API 路由
9. 启动服务器

### 6.2 API 请求处理流程

1. 接收 HTTP 请求
2. 应用全局中间件（日志、错误处理、速率限制）
3. 路由匹配
4. 应用路由中间件（认证、权限）
5. 执行处理器逻辑
6. 返回响应

### 6.3 认证流程

1. 用户发送登录请求
2. 验证用户凭证
3. 生成 JWT 令牌
4. 返回令牌和用户信息
5. 后续请求携带令牌
6. 中间件验证令牌有效性
7. 提取用户信息并注入到请求上下文

### 6.4 权限验证流程

1. 请求到达需要权限的接口
2. 认证中间件验证用户身份
3. 权限中间件检查用户是否拥有所需权限
4. 允许或拒绝访问

## 7. 技术实现细节

### 7.1 配置管理

- 使用 YAML 格式的配置文件
- 支持不同环境的配置（开发、生产、测试）
- 配置自动加载和解析

### 7.2 数据库管理

- 使用 GORM 作为 ORM 框架
- 自动迁移数据库模型
- 初始化默认数据

### 7.3 缓存管理

- 使用 Redis 作为缓存
- 缓存连接失败不影响服务启动
- 用于提高系统性能

### 7.4 安全措施

- 密码加密存储
- JWT 令牌认证
- CORS 配置
- 安全头部设置
- 速率限制防止暴力攻击

### 7.5 错误处理

- 统一的错误响应格式
- 错误日志记录
- 友好的错误提示

## 8. 系统监控与维护

### 8.1 日志系统

- 请求日志记录
- 错误日志记录
- 系统运行状态日志

### 8.2 健康检查

- 提供健康检查接口
- 监控系统运行状态

### 8.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 数据库连接失败 | 数据库服务未运行或配置错误 | 检查数据库服务状态和配置 |
| Redis 连接失败 | Redis 服务未运行或配置错误 | 检查 Redis 服务状态和配置 |
| 令牌无效 | JWT 密钥配置错误或令牌过期 | 检查 JWT 配置和令牌有效期 |
| 权限不足 | 用户角色或权限配置错误 | 检查用户权限配置 |
| 速率限制 | 请求频率过高 | 合理控制请求频率 |

## 10. 总结

Hermes Platform 是一个功能完整的测试管理平台，提供了测试任务、测试详情和测试记录的全生命周期管理。系统采用分层架构设计，具有良好的扩展性和可维护性。通过 JWT 认证和基于角色的权限控制，确保系统的安全性。系统支持容器化部署，便于在不同环境中快速部署和运行。

未来可以通过扩展功能模块、优化性能和增强安全性，进一步提升系统的价值和用户体验。
